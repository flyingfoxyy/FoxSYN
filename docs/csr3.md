# csr3 — 割线即编码：用 ACD 视角回收跨 FPGA 冗余线

> 这是一份方法论 + 设计文档，描述 `csr`/`csr2` 之后的第三代思路。`csr`/`csr2` 把跨分区边（cut-edge）当**结构问题**，靠 resub 和 replication 逐条消边；`csr3` 换一个视角——把一整束割线看成一个**多输出函数的编码**，用信息论下界衡量它浪费了多少，再决定要不要重编码。两者不冲突，`csr3` 看得见前两者看不见的东西：**一束线的联合冗余**。
>
> 本文尚无实验数据。它的目的是把方法讲清楚、把每一步的正确性论证钉死、把"先测量后实现"的门控（gating）固化成流程，以便据此立项。

## 1. 背景与动机

### 1.1 割线为什么贵

硬件仿真器（emulator / prototyping）把一个超大设计切成多块 FPGA 协同运行。分割后，一个信号如果它的驱动者（driver）和消费者（consumer）落在不同 FPGA 上，就必须走**板间物理连线**。板间引脚数量远小于片内布线资源，于是所有跨界信号被 **TDM**（Time-Division Multiplexing，时分复用）挤到有限的物理通道上分时传输：k 根逻辑割线复用 1 根物理线，就要花 k 个时间片才能传完一拍的数据。

> **TDM ratio**：一根物理线上复用的逻辑信号数。ratio 越高，仿真主频越低——因为一个仿真周期必须等最慢的那束线传完。整机速度由**最坏的那对板子**（peak board-pair）的 cut 决定，不是平均。

结论：跨界线是仿真器里**最贵的资源**。少一根跨界线，可能就少一个 TDM 时间片，直接换成主频。相比之下，片内多几个 LUT 几乎不要钱。这个"线贵、逻辑便宜"的成本比，是本方法所有取舍的出发点。

> **LUT**（Look-Up Table，查找表）：FPGA 的基本逻辑单元，一个 k 输入 LUT 能实现任意 k 变量布尔函数。`if -K 6` 就是把网表映射成 6 输入 LUT。
>
> **FF**（Flip-Flop，触发器 / 寄存器）：时序单元，在时钟沿锁存数据。它的输出端习惯记作 **Q**。FF 把电路切成一段段纯组合逻辑。

### 1.2 现有优化命令的盲区

FoxSYN 里已有几个跟分区打交道的命令：

- **`cpr`**：relocate / replicate 节点降 **hop 数**（timing）。
- **`cmfs`**：SAT-based resub 优化关键路径 **arrival**（timing）。
- **`csr` / `csr2`**：直接以 **cut-edge** 数量为优化目标，逐条消跨分区边。

它们的共同前提是**逐节点、逐边**地看问题：resub 换一个 fanin、replication 复制一个 driver。这类局部动作看不见一个全局事实——**一束线作为整体，可能携带的信息量远小于它的位宽**。

举个极端例子：A 侧一个 32 状态的 one-hot 状态机，32 根状态线全部过界给 B。每根线单独看都不冗余（都真的在变化），resub 找不到任何一根能被同分区 divisor 替换。但这 32 根线**联合起来**只有 32 种合法取值，真正的信息量是 `⌈log₂ 32⌉ = 5` 位——27 根线是纯粹的浪费。`csr`/`csr2` 对此完全无能为力，因为冗余不在任何单条边上，而在**一束线的联合结构**里。

### 1.3 视角转换

`csr3` 的全部出发点就是这一句：

> **割线优化不是网表结构问题，是信息编码问题。** 划分器和 resub 看不见"一束线的联合冗余"，因为它们没有把这束线当成一个整体函数来问"它到底需要多宽"。

一旦这样问，问题就落进了逻辑综合里一个有 60 年历史的经典框架——**Ashenhurst-Curtis 分解**。

## 2. 核心洞察：割线即编码

### 2.1 一束割线就是一个多输出函数

取 A→B 方向的一束 k 根割线。每根线的值都由 A 侧的某些信号决定。把这些决定它们的 A 侧信号收集起来，记作 **support**（支持集）。那么这束线就是一个多输出布尔函数：

```
h : {0,1}^|support|  →  {0,1}^k
```

`h` 已经存在——它就是 A 侧那几百个 LUT 组成的组合逻辑。**它是一个现成的编码**：把 support 上的信息编成 k 位送过界。问题在于，这个现成编码**几乎肯定是浪费的**——它是综合器为了实现功能顺手产生的，从没有人以"用最少的线传递等量信息"为目标优化过它。

### 2.2 当前宽度 k vs 信息论下界

信息论给出一个硬下界：如果 B 侧最终只能区分出 μ 种不同的 A 侧行为，那么过界只需要传 `⌈log₂ μ⌉` 位就够了。于是：

```
当前编码宽度   = k                （现在真的用了几根线）
信息论下界     = ⌈log₂ μ⌉         （理论上最少几根线）
可回收物理线   = k − ⌈log₂ μ⌉     ← 这就是"水分"
```

`csr3` 要做的事，一句话：**把现成但低效的编码 `h`，换成一个宽度逼近 `⌈log₂ μ⌉` 的新编码**。A 侧装一个 **encoder**（把宽的编成窄的），B 侧装一个 **decoder**（把窄的还原成 B 需要的），中间过界的线数从 k 降到接近 `⌈log₂ μ⌉`。

### 2.3 μ 从哪来：两层 don't care

宽度能压下来，是因为存在 **don't care**（无关项）——某些输入组合"不会发生"或"发生了也没人看"，综合时可以随意处置。水分来自两类 don't care：

| 来源 | 含义 | 对应量 |
|---|---|---|
| **SDC**（Satisfiability Don't Care，可满足性无关项） | A 侧**根本产生不出**的 k 位组合 | 可达组合数 `m ≤ 2^k` |
| **ODC**（Observability Don't Care，可观测性无关项） | B 侧**行为无差别**、可以合并的组合 | 等价类数 `μ ≤ m` |

> **don't care**：一个布尔函数在某些输入下的取值"无所谓"——要么这个输入永不出现（SDC），要么它的输出无人观测（ODC）。综合器可以把 don't care 填成任意值以简化逻辑，这是逻辑优化最主要的自由度来源。

链条是 `μ ≤ m ≤ 2^k`：

- 从 `2^k` 到 `m`：k 根线联合起来实际只能取到 m 种值（比如 one-hot 只有 k 种，不是 `2^k` 种）。这是**可达性 / SDC** 水分。
- 从 `m` 到 `μ`：这 m 种可达组合里，有些 B 侧压根分不清（导致相同行为），可以进一步合并成 μ 个等价类。这是 **ODC** 水分。

下界是 `⌈log₂ μ⌉`。但 μ 要动 B 侧、成本高。**一期只做 m**（`⌈log₂ m⌉`），它便宜、收益已经可观，μ 留作二期。

## 3. 与经典 ACD 的对应关系

### 3.1 什么是 ACD

> **Ashenhurst-Curtis 分解（ACD）**：逻辑综合里的一种函数分解方法。给一个函数 `f(X)`，把变量集 `X` 切成两块——**bound set**（约束集 `X_b`）和 **free set**（自由集 `X_f`），寻找一种形式：
>
> ```
> f(X) = g( h₁(X_b), …, h_t(X_b),  X_f )
> ```
>
> 即先用 t 个中间函数 `h₁…h_t` 把 bound set "压缩"成 t 位，再由 `g` 结合这 t 位和 free set 算出结果。t 的最小值是 `⌈log₂ μ⌉`，其中 **μ 是分解表（decomposition chart）的列重数（column multiplicity）**——把真值表按 bound set 排行、free set 排列，本质不同的列有多少种。列重数越小，t 越小，中间需要的连线越少。

ACD 的原始动机就是**减少子函数之间的连线**——这和割线优化想干的事，本质是同一件事。

### 3.2 映射到分割场景

把 ACD 的每个部件对号入座：

```
bound set  X_b   =  A 侧 support（FF 输出 Q + A 的输入端口 + 常量回溯到的信号）
h 函数     h₁…hₜ =  现有的 k 根跨界线（已经是一个分解，只是 t=k 编码浪费）
free set   X_f   =  B 侧自己的其他输入（可作 side information，二期用）
列重数     μ     =  B 侧行为等价类数
t（新宽度）      =  我们要造的 encoder 输出位数，目标 ⌈log₂ μ⌉
我们做的事       =  把现成但低效的 h（宽度 k）换成宽度 ⌈log₂ μ⌉ 的新 h（= encoder），
                    B 侧补 g 的对应部分（= decoder）
```

一句话对应：**现有的 k 根割线 = 一个已经存在、但编码宽度远超列重数下界的 ACD 中间信号 `h`。** `csr3` 就是重新解这个分解，把 t 从 k 压到 `⌈log₂ μ⌉`（一期先到 `⌈log₂ m⌉`）。

## 4. 两个压缩来源与守恒律

这一章把整套方法赖以成立的**正确性论证**集中讲清楚，后面每一步都靠它。

### 4.1 support 当自由变量 ⇒ 收益是下界，无条件安全

抽锥（cone extraction）时，我们把 support 里的每个信号都当作**自由布尔变量**——即假设它能独立取 0/1 的任意组合，不管 A 侧上游其实有没有约束把它们绑在一起。

这个假设是**故意偏保守**的，它带来一条守恒律：

> 把 support 当自由变量算出的可达组合数 m，是**真实可达数的上界**（真实上游约束只会让可达组合更少，不会更多）。于是 `k − ⌈log₂ m⌉` 是**真实可回收线数的下界**。据此发现的每一分收益都无条件正确，不依赖任何时序假设；漏掉的（真实 m 其实更小）留给后续迭代。

**永不过报**是这套方法能安全落地的根基：Phase 0 说有 3 根水分，那就一定至少有 3 根；它可能说 0（其实有），但绝不会说 5（其实只有 3）。

### 4.2 三台仪器都是单侧下界

Phase 0 里有三个测量装置，全部是**保守的下界仪器**，方向一致：

| 仪器 | 测什么 | 为什么偏保守（低估水分） |
|---|---|---|
| **m 计数**（SDC） | 组合可达性 | support 当自由变量 ⇒ m 偏大 ⇒ 水分偏少 |
| **不变式探针**（可达性） | 时序不可达组合 | 1-induction 不完全 ⇒ 只 credit 证得动的那部分 |
| **μ / ODC** | B 侧可合并 | 二期才做 ⇒ 一期完全未计 |

三台仪器的读数叠加，仍然是真实水分的**下界**。因此：

> Phase 0 报表的口径必须是 **detected-floor（探到的下限）**，不是 **zero（真没水）**。"两列都≈0"严格只能读作"三台仪器都探不到"，绝不能读作"这束线没有冗余"。措辞错了会误导决策——尤其是 one-hot 这类组合层恒盲的场景（见 [第 10 章](#10-标杆场景one-hot-总线过界)）。

### 4.3 仿真级联的安全性

Phase 0 用"随机仿真 → SAT"两级级联定 m。仿真数出的 distinct 组合数是 m 的**下界**（没跑到的组合不会被数到）。安全性论证是**"误杀不可能 + 误留被兜住"**，不是"不误留"：

- `仿真 distinct > 2^(k-1)` → **淘汰**。下界都已过阈值，真值必过，**永不误杀**（不会砍掉真有水分的组）。
- `仿真 distinct ≤ 2^(k-1)` → **放行**。欠采样时，本该淘汰的组可能**误留**进 SAT 层——但 SAT 枚举超过 `2^(k-1)` 就早退，只损失一点时间，不损失正确性。

误留无害，误杀不可能。这就是级联可以放心用仿真做粗筛的理由。

## 5. 总体架构：测量先于实现

整个流程分两个 Phase（μ/ODC 是二期，记作 Phase 2）：

```
Phase 0  水分测量   —— 只测不建，用一个数字回答"值不值得做"
   │
   │  gate: 峰值板对水分 × TDM 量化台阶
   ▼
Phase 1  重编码     —— 真正的 encoder/decoder 引擎，逐组独立、并行、失败跳过
   │
   │  gate: 前两层榨干后峰值仍卡台阶 + B 侧有明显 don't care 结构
   ▼
Phase 2  μ/ODC      —— 动 B 侧扇出锥，逼近真正的 ⌈log₂ μ⌉（二期）
```

**为什么必须先测后建**：实现成本（综合 encoder/decoder + CEC + 时序回灌）比测量成本高一个数量级。倒着做——先花大力气实现、再发现根本没水分——是最典型的浪费。这个 gating 本身就是方法论的一部分，不是可选的工程优化。测量先行的另一个价值：**它能主动阻止无效投入**——如果 Phase 0 报告结构性水分≈0，结论是"该转向 TDM 削峰或改划分目标函数"，这个"否定"结论本身就省下了几个月投在错方向上的工。

## 6. Phase 0：水分测量

七步，从整机分割结果一路收敛到一张裁决报表。

### Step 1 — 切束

从分割结果中拿出所有 A→B 有向信号束。**切分粒度是 `(源分区, 目的分区, 方向)` 三元组**：每对板子、每个方向独立成束、独立处理。理由见 [§9.3](#93-多目的地扇出为什么独立处理)。

### Step 2 — 抽锥，界定 support

对束里每根线，向 A 内部做**传递扇入（transitive fanin）**回溯，停止条件：碰到 FF 的 Q 端、A 的输入端口、或常量。回溯经过的组合逻辑就是这根线的**锥（cone）**（通常几百个 LUT），锥的输入就是它的 **support**。

> **cone（锥）/ transitive fanin**：从一个信号往输入方向一路收集所有影响它的逻辑，直到边界（这里是 FF/端口/常量）。锥是一段纯组合逻辑，因此可以用真值表 / 仿真 / SAT 精确刻画。

**关键**：support 一律当自由布尔变量（[§4.1](#41-support-当自由变量--收益是下界无条件安全)）。算出的 m 是上界，据此发现的压缩无条件安全，不依赖任何时序假设。

### Step 3 — Jaccard 聚类分组

联合冗余只在 **support 重叠**的线之间存在——两根 support 完全不相交的线，它们联合的可达组合数 `m = m₁ × m₂`（乘积），放一组没有任何联合收益。据此分组：

1. 每根线的 support 用 bitset 表示。
2. 两两算 **Jaccard 相似度** `J = |support_i ∩ support_j| / |support_i ∪ support_j|`。
3. `J > 阈值（~0.3）` 连边。
4. 连通分量内部再切成 `k ≤ 8~16` 的小组（k 是每组的线数，也是待压缩的位宽上界，太大 SAT 吃不消）。

> **Jaccard 相似度**：两个集合的交并比，衡量重叠程度，值域 [0,1]。这里用来判断两根线是否"看同一批上游信号"。

这一步把搜索空间从组合爆炸砍到**线性**：不再枚举所有线的子集，只在 support 高度重叠的小簇内找联合冗余。

### Step 4 — 随机仿真预过滤

每组的锥做 **bit-parallel 软件仿真**（一个机器字并行跑 64 个随机向量），随机灌 support，用 hash set 数 k 位输出的 distinct 组合数。

> **bit-parallel 仿真**：把 64 个独立输入向量塞进一个 64-bit 字的不同位，一次按位运算同时算完 64 组，是软件模拟布尔电路最快的手法。

判据（[§4.3](#43-仿真级联的安全性)）：`distinct > 2^(k-1)` → 无压缩空间，**淘汰**。经验上这一步能淘汰 90%+ 的组，把昂贵的 SAT 只留给真正可疑的少数。

### Step 5 — SAT 枚举定 m

幸存组的锥转成 **CNF**，做投影 **All-SAT**：

> **CNF**（合取范式）/ **SAT**：把电路编码成布尔可满足性问题的标准输入格式。SAT 求解器判定"是否存在一组输入让公式为真"。
>
> **All-SAT + blocking clause（阻塞子句）**：要枚举所有不同的输出组合，每求得一个解，就往公式里加一条子句禁止**这个输出组合 y** 再次出现（只对输出变量 y 投影加 blocking，不管 support 怎么取），继续求解，直到 UNSAT。解的个数就是可达组合数 m。

优化：一旦枚举数超过 `2^(k-1)` 就**早退**（已经证明没有压缩价值，不必数到底）。这让成本和价值天然对齐——**m 小的组恰好最便宜就能定**，而 m 小正是我们要的。

### Step 6 — 不变式探针（并行第二只眼，不等 m 报表）

Step 4/5 是**组合层**仪器，对**时序可达性**水分完全盲。最典型的盲区：one-hot 状态机过界时，support 就是状态 FF 本身、锥是平凡直连，`m = 2^k`，组合仪器读数恒为 0——可真实可达只有 k 个 one-hot 码字，藏着巨大水分。

所以引入第二台仪器，**独立触发、不等 m 报表**（因为它几乎免费）：

1. **候选生成**（两条来源，互补）：
   - **结构信号**：k 根线回溯到浅锥 + support 以 FF 为主 + 这些 FF 属于同一寄存器组（同 always 块 / 同 enable / 前端 FSM 提取信息 / one-hot 综合属性）→ 高度可疑。注意：**"深组合锥→SDC 水分" 和 "浅锥直连 FF→可达性水分" 正好覆盖互补区域**，`m≈2^k` 加浅锥 FF-support 本身就是最强触发特征。
   - **仿真信号**：从**复位态**出发做 A 的时序随机仿真（复位可达状态 ⊆ 真可达状态，拿来挖候选是 sound 的），对过界线套模板：**one-hot / 互斥（mutex）/ 常量位 / Gray 码**。
2. **1-induction 证明**：把候选不变式送去归纳证明。

   > **1-induction（1-归纳）/ inductive invariant**：证明一个性质 P 对所有可达状态成立的一种方法——证 (a) 复位态满足 P，(b) 任意满足 P 的状态走一拍后仍满足 P。成立则 P 是不变式。**它 sound 但不完全**：证不动不代表 P 不成立。
   >
   > 尤其 one-hot：逐条互斥 `¬(sᵢ ∧ sⱼ)` **不是** 1-归纳的（只假设单条、放开其余，下一拍拦不住别的畸形态里两位同时置起）；必须把整个 one-hot 约束当成**合取（conjunction）整体**做**归纳强化（strengthening）**才证得动。

3. **回灌重算**：证明成立的不变式作为约束加进该组的 SAT 枚举，重跑 m，得到收紧后的 **m′**。

因为 1-induction 不完全，这台仪器和 m 计数**同性质，也是单侧下界**——只 credit 证得动的那部分可达性（[§4.2](#42-三台仪器都是单侧下界)）。

### Step 7 — 报表 + 裁决

报表按板对组织（每个组天然属于某个有向板对），出**三个数**，各司其职：

1. **全局和** `Σ_group (k − ⌈log₂ m′⌉)` —— **存在性粗筛**。它只回答"这个现象在我们的设计上到底有没有"，是给**方法**定生死的，不是给**项目**定 go/no-go 的。
2. **峰值板对水分 × TDM 量化台阶** —— **真正的 go/no-go**。理由见 [§9.4](#94-裁决为什么看峰值板对而非全局和)。判据：
   ```
   gain_effective = TDM_ratio(cut_now) − TDM_ratio(cut_now − water_peak)
   ```
   削峰是**迭代**的：压掉当前峰值后瓶颈转移到次峰，有效收益 = `old_max − new_max`，所以要看整条**排序后的 cut profile**，不是单点。好在 Phase 0 数据全在手，这个迭代是纯报表运算，免费。
3. **按"是否压在临界 cut 上"排序的 group 清单** —— **Phase 1 的工单顺序**。非峰值板对的压缩降级为"省布线资源 / 缓解拥塞"的二级收益，默认不做。

三种结局：

- **两列水分（组合 m′ + 不变式）都 ≈ 0**（读作 detected-floor，非 zero）→ 结构性优化到头，转向 TDM 削峰 / 改划分目标函数。**这个否定结论本身就是成果**——它阻止了投在错方向上的工。
- **水分集中在少数束** → 只做定点重编码，或把这些束反馈给前端改 RTL / 分区约束。
- **水分普遍且大** → 立项 Phase 1。

## 7. Phase 1：重编码实现

逐组独立、天然并行、失败直接跳过（不影响其他组）。四步。

### Step 8 — 码字分配（code assignment）

知道 m′ 个可达组合后，要给它们各分配一个 `t = ⌈log₂ m′⌉` 位的**码字**。这是 ACD 的经典难题：**编码的选择决定了 encoder/decoder 的逻辑成本**，是 NP-hard。实用启发式：

- **优先选"A 网里已有现成信号能当码位"的编码**（见 Step 9 的机会主义匹配）——把编码自由度花在"白嫖现成信号"上，比花在化简逻辑上更值。
- **尽量保留原始位**：如果某根原线本身能把组合集近似均分，保留它当码字的一位，decoder 那位就是直连线。
- **未用码字**（`2^t − m′` 个）全部当 don't care 喂给两侧综合。
- **相邻输入组合给相邻码字**（汉明距离小），利于逻辑化简。

不追求最优——够用即可。

### Step 9 — 综合 encoder / decoder（锥不动）

> **锥保持不动**：A 侧那几百 LUT 的锥**原样保留**。新增的只是作用在 k 根线上的 `k→t` **压缩器（compressor / encoder）**（在 A），和 `t→k` **解压器（decompressor / decoder）**（在 B）。过界的只有 t 根。省 LUT 不是目标，**省线才是**；动锥会把 CEC 和时序扰动范围扩大一个数量级，得不偿失。因此 truth table / ISOP 只在 `k↔t` 这一层做（`2^k`，`k ≤ 16` 可行），**不是**在 `support→t` 上做（那要 `2^|support|`，几百输入直接爆）。

**机会主义 encoder（关键增益点）**：纯 `k→t` 综合会错过一类**免费**的 encoder，恰好是最值钱的场景。如果 k 根线是某个二进制状态 decode 出来的 one-hot，理想 encoder 根本不用综合——码字就是 A 里**已经存在**的那几个二进制状态 FF，直接引出来即可，零逻辑。纯 `k→t` 视角只会给你造一个 **priority encoder**（优先编码器），功能对但白花 LUT 和延迟。

> **priority encoder（优先编码器）**：把 one-hot / 多热输入转成二进制序号的组合逻辑。功能上能当 encoder，但它是"造"出来的逻辑，而机会主义匹配是"白嫖"已有信号。

所以 Step 9 加一个廉价步骤：选定码字后，对每个码位问"**A 网表里是否已存在一个信号恰好等于它（或其反）**"——**仿真 signature 匹配预筛 + 一次 SAT 确认**，命中就**直接引线、零逻辑**，未命中的码位才落到 `k→t` 压缩器里。这不是从 support 重新综合（不需要 `2^|support|` 的表），只是在已有网里做一次 **resub 式查找**，成本可忽略。

> **signature（仿真签名）**：一个信号在一批随机向量下的输出位串。两个信号 signature 不同则必不等价（快速否证）；signature 相同再用一次 SAT 确认真等价。这是 resub 里标准的"先仿真筛、再 SAT 证"套路。

**代价关卡**：若 encoder + decoder 的 LUT / 深度代价超过省下的线的价值，放弃该组。线在 emulator 里极贵，通常不会触发，但门要留着。

### Step 10 — 等价性验证（CEC）

每组做 **cone-level CEC（组合等价性检查）**。

> **CEC / miter**：证两个组合电路功能等价的标准方法。搭一个 **miter**——把两个电路同输入并联、输出异或，再问 SAT "异或能否为 1"。UNSAT 则处处相等，等价。

**miter 构造的关键细节**：**不能**验 `decoder(compressor) == identity`（对全 `2^k` 输入）——那会把 don't care 全判成错（不可达组合上，压缩再解压本来就不必等）。正确的 miter **把锥包进来**：

```
∀ x ∈ support :   decoder( compressor( cone(x) ) )  ==  cone(x)
```

锥的存在恰好把输入限制在**可达像**上，don't care 的合法性由此**免费获得**——这也是"锥不动"的第二个理由：**锥就是 CEC 的假设生成器**。锥小，秒级。这一步**不可省**，它是全流程的安全网。

### Step 11 — 集成回流程

- **插入点**：分割之后、TDM 分配 / 布线之前。
- **省下的线优先削峰值 cut**（决定 TDM ratio 的是最坏板对），不是摊平均。
- **feed-through 免费兼容**：若某束物理上走 A→B→C 中转，只要 C 消费整束、B 不消费，则 B **只透传 t 根码线、不挂 decoder**——码字过境不解码。这是该方案对多跳路由天然友好的一点。B 也消费时才在 B 挂 decoder。
- **时序核算**：encoder / decoder 各加 1~2 级 LUT 延迟；但跨界线走 TDM 时延迟以数百 ns 计，这点组合深度增加通常可忽略。**仍需回灌时序引擎确认**，不能默认忽略。

## 8. Phase 2：μ/ODC（二期）

一期把 m（组合 SDC + 已证的可达性）榨干后，若峰值板对**仍**卡在 TDM 台阶上，才考虑最后一层：从 m 逼近真正的下界 μ，回收 **ODC** 水分。

**触发条件（独立 gate，比 Phase 1 严）**：
1. SDC + 不变式做完之后，峰值板对**仍**卡在 TDM 台阶；且
2. 该板对的束在 **B 侧有明显的 don't care 结构**（比如 B 只在特定使能下才采样这些线）。

**为什么单列二期**：ODC 要动 **B 侧扇出锥**，CEC 粒度从 wire-level 放松到 **output-level**（decoder 不必还原 k 根原线，只需保住 B 的可观测行为），验证成本贵一档。它是"前两层榨干后仍不够"时的**最后一层**，不是默认路径。

**side information**：Phase 2 还可利用 free set（B 侧已知信号）——support 中 B 也看得到的信号做**条件计数**，传条件熵而非无条件熵，进一步压宽度。同属二期。

## 9. 几个关键设计决策的论证

把散在流程里、但值得单独钉死的设计取舍集中说明。

### 9.1 为什么锥不动

省 LUT 不是目标，省线才是。锥不动带来三个好处：(1) 成本可控——新增逻辑只在 `k↔t` 这层，`2^k` 可算；(2) CEC 便宜——锥限定了可达像，don't care 合法性免费；(3) 时序扰动范围最小。动锥会把三者全部扩大一个量级，而换来的 LUT 节省在"线贵逻辑便宜"的成本模型下几乎无意义。

### 9.2 miter 为什么必须包锥

见 [Step 10](#step-10--等价性验证cec)。核心：验 identity 会误判 don't care 为错误；包锥把输入约束到可达像，才是正确的等价性问题。

### 9.3 多目的地扇出为什么独立处理

一根 A 侧信号同时去 B 和 C，当成**两个独立 bundle**（A→B、A→C）各自分组、各自编码。同一信号在不同方向可拿**不同码字**，压缩器**不跨目的地共享**。

- **为什么不共享**：LUT 相对线是白菜价，跨方向共享压缩器的工程复杂度不值这点逻辑节省。
- **不重复计数**：两个方向占各自的物理通道，各自的 gain 各自真实，不存在重复。
- **数据可复用**：同一信号进了两个方向的组，**锥抽取和仿真数据可以复用**，只有 SAT 枚举和编码按方向独立做。

### 9.4 裁决为什么看峰值板对而非全局和

TDM ratio 由**最坏板对**定。于是会出现两种失配：全局水分很大却都落在非峰值板对（削不动 max cut → 不改善主频），或全局水分不大却正好压在峰值板对（极值的钱）。所以：

- **全局和** = 存在性粗筛（方法生死）。
- **峰值板对水分 × TDM 量化台阶** = 项目 go/no-go。而且要过 **TDM 量化台阶**——ratio 是受 SerDes 帧格式约束的**离散台阶**，峰值板对省 3 根线若不足以跨过一个台阶，物理收益是**零**。

> **SerDes / TDM 量化台阶**：板间高速串行收发器按固定帧格式打包。TDM ratio 只能取离散值（如 4、8、16），落在两个台阶之间的线数节省不产生主频收益。所以判据必须是"是否跨过台阶"，不是"省了几根线"。

### 9.5 为什么不变式探针要提前进 Phase 0

原始设想里，时序不变式是 Step 11 的二期增强。**修正：提前进 Phase 0，作为第二只眼**。理由：(1) 触发器几乎免费，每组秒级，Phase 0 完全养得起；(2) 它和 m 报表**互补覆盖**（深锥 SDC vs 浅锥可达性），少了它，one-hot 这类"最值钱"的水分会在 Phase 0 恒盲、被当成 0 收益丢掉；(3) "到头"这个否定结论必须两台仪器都探不到才敢下，否则会误导决策转向。

## 10. 标杆场景：one-hot 总线过界

把前面几条串起来，one-hot 过界是**"隐形 × 最值钱 × 最便宜"三合一**，应当排在 Phase 1 工单**最前**——它恰好是 naive 方法的最大盲区，又是净收益最高、实现最轻的靶子。

设 A 侧一个 k=32 的 one-hot 状态机，32 根状态线过界给 B：

| 维度 | 表现 | 涉及机制 |
|---|---|---|
| **隐形** | 组合层 `m = 2^k`，Step 4/5 读数为 0，naive 方法直接当无收益丢弃 | 组合仪器对可达性盲（[§4.2](#42-三台仪器都是单侧下界)） |
| **被救回** | Step 6 结构信号（浅锥 + FF-support + one-hot 属性）触发，1-induction 证 one-hot 不变式（须合取强化），重算得 `m′ = 32` | 不变式探针（[Step 6](#step-6--不变式探针并行第二只眼不等-m-报表)） |
| **最值钱** | 真实只可达 32 个码，该传 `⌈log₂ 32⌉ = 5` 位，**省 27 根线** | 可达性水分 |
| **最便宜** | 理想 encoder = A 里已有的 5 个二进制状态 FF，直接引线、**零逻辑**；decoder = 把当年 A 侧的 decode 挪到 B | 机会主义 encoder（[Step 9](#step-9--综合-encoder--decoder锥不动)） |

关键在于：**同一个结构签名**（浅锥 + FF-support + one-hot 属性）同时触发了 [§9.5](#95-为什么不变式探针要提前进-phase-0) 的第二只眼（把这组从 m 滤网里救回来）**和** Step 9 的免费 encoder（白嫖已有 FF）。测量端和实现端的最大机会，指向同一个结构特征。

## 11. 术语表

| 术语 | 含义 |
|---|---|
| **LUT** | Look-Up Table，FPGA 基本逻辑单元，k 输入 LUT 实现任意 k 变量布尔函数 |
| **FF / Q 端** | Flip-Flop 触发器；Q 是其输出端，锥回溯的停止边界之一 |
| **cut / cutsize** | 跨分区的连线集合 / 其数量 |
| **cut-net vs cut-edge** | net：一条 net 有任意 fanout 跨区就记一次；edge：每对跨区 (driver, consumer) 各记一次 |
| **fanin / fanout** | 一个节点的输入信号 / 输出去向 |
| **cone（锥）/ transitive fanin** | 从一个信号往输入方向收集到边界的全部组合逻辑 |
| **support（支持集）** | 一个函数/锥的输入变量集合 |
| **don't care** | 输入取值"无所谓"的项，逻辑优化的自由度来源 |
| **SDC** | Satisfiability Don't Care，可满足性无关项——永不出现的输入组合 |
| **ODC** | Observability Don't Care，可观测性无关项——输出无人观测的组合 |
| **ACD** | Ashenhurst-Curtis Decomposition，把函数按 bound/free set 分解、以最少中间连线实现 |
| **bound set / free set** | ACD 中被压缩的变量集 / 直接透传的变量集 |
| **column multiplicity（列重数 μ）** | 分解表中本质不同的列数，决定中间连线下界 `⌈log₂ μ⌉` |
| **m** | 一束线联合的可达组合数，`μ ≤ m ≤ 2^k` |
| **encoder / decoder** | A 侧压缩器（`k→t`）/ B 侧解压器（`t→k`） |
| **priority encoder** | 把 one-hot 转二进制序号的组合逻辑；机会主义匹配想避开的"造出来的逻辑" |
| **truth table / ISOP** | 真值表 / Irredundant Sum-of-Products，不可约积之和，逻辑综合中间表示 |
| **resub（resubstitution）** | 用网表里已有的其他信号（divisor）替换一个节点的实现 |
| **signature（仿真签名）** | 信号在一批随机向量下的输出位串，用于快速否证等价 |
| **SAT / CNF** | 布尔可满足性问题 / 其标准合取范式输入 |
| **All-SAT / blocking clause** | 枚举所有解 / 每得一解加子句禁止其再现 |
| **CEC / miter** | Combinational Equivalence Checking / 两电路输出异或后判 SAT 的等价性检查结构 |
| **Jaccard 相似度** | 两集合交并比 `|∩|/|∪|` |
| **bit-parallel 仿真** | 一个机器字并行跑多个随机向量的软件仿真 |
| **1-induction / inductive invariant** | 单步归纳证不变式，sound 但不完全 |
| **one-hot** | 编码方式，任意时刻恰有一位为 1 |
| **TDM** | Time-Division Multiplexing，多根逻辑割线分时复用一根物理线 |
| **TDM ratio / 量化台阶** | 复用比；受 SerDes 帧格式约束只能取离散值 |
| **SerDes** | 板间高速串行收发器，TDM 的物理载体 |
| **feed-through** | 信号途经中转 FPGA 但该 FPGA 不消费，只透传 |

## 12. 后续方向

- **μ / ODC 全量实现**（Phase 2）：动 B 侧扇出锥做等价类合并，逼近真正的 `⌈log₂ μ⌉`。
- **side information**：把 free set（B 侧已知信号）作条件，传条件熵而非无条件熵。
- **不变式模板扩充**：one-hot / mutex / 常量 / Gray 之外，挖更多可 1-induction 证明的时序不变式模板。
- **可压缩性反馈划分器**：把"可压缩性"作为权重喂回 min-cut，让划分倾向于从可压缩的束上切。记为 future work，别进 v1。
- **与 `cpr`/`cmfs`/`csr` 协同**：csr3 降的是"束的编码宽度"，与逐边 resub / relocation 正交，可叠加迭代。
